核心命題：標準取代平台

2026 年的 agent 不是透過某個 app 找到你的網站，而是透過開放網路標準直接與你的服務互動。
過去

企業仰賴平台 app 與整合，受平台政策、審核、抽成牽制。

2026

企業實作開放協定，agent 直接讀取結構化資料、發起交易。

01

商務層

Commerce / UCP

02

編輯判斷可被編碼，隨工具自動更新到所有客戶端。

路線之爭：開放標準 vs 平台主導

UCP / MCP / Skills

開放網路標準
  • 企業獨立參與
  • 無需 app 或審核
  • 多廠商共治
  • 標準制定就是槓桿

OpenAI ACP

Agentic Commerce Protocol
  • 單一平台主導
  • 需經平台審核
  • 政策可能單方面變更
  • 抽成與排序由平台決定

對新聞媒體的三個啟示

01

實作開放網路標準，不要單押平台整合

把 server-card、UCP endpoint 視為跟 RSS、sitemap 同等級的基礎設施。

02

推動標準涵蓋知識產品與訂閱

UCP 目前只覆蓋實體 SKU。新聞、付費內容、訂閱模式還沒納入，現在進場仍能影響規格。

03

用 Skills 編碼編輯判斷，建立差異化

「給解釋性報導再給即時新聞」這類編輯邏輯，是 agent 環境下的競爭優勢。
